Hasil saya menyadur materi dan penjelasan video pada pertemuan ke 6:
  1. Menjelaskan mengenai class diagram beserta pengertian dan penggunaan dari class diagram yang mengambarkan hubungan antar kelas atau antar tabel
  2. Menjelaskan class diagram merupakan representasi lain dari Entity Relationship Diagram (ERD) dan Logical Record Structure (LRS), tetapi lebih lengkap karena memuat struktur tabel dan operasi terhadap basisdata (query) terhadap tabel tersebut.
  3. Class diagram terdiri dari : nama class, attribut, method dimana memiliki fungsi yang sangat penting dalam membuat sebuah rancangan yang membentuk class diagram.
  4. Stereotype (dalam basis data) yaitu menentukan / menandakan attribut yang menjadi pk = primary key maupun fk = foreign key.
  5. Multiplicity yaitu menentukan tingkat hubungan / kardinalitas antar class melalui association (garis hubungan antar class), yang ditentukan dengan tingkatan sebagai berikut:
    1. 1 => hanya 1 dan pasti hanya 1
    2. 0..1 => dapat tidak ada (kosong) sampai dengan 1
    3. 0..* => dapat tidak ada (kosong) sampai dengan tak terhingga (banyak)
    4. * => dapat banyak (tak terhingga)
    5. 1..* => sedikitnya 1, bisa lebih dari 1
  6. Class diagram menggambarkan hubungan antar class atau hubungan antar tabel (struktur tabel)
  7. Method diisi dengan operasi apa saja yang dilakukan pada class/tabel, misal: select/baca,insert/tambah/simpan,update/ubah,delete/hapus,count/hitung jumlah, auto_numbering/nomor otomatis/kode otomatis, dst.
